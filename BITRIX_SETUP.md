# Регистрация приложения в Bitrix24

Это руководство для регистрации Expo Calendar в вашем портале Bitrix24.

## 1. Подготовка

Убедитесь, что у вас есть:
- [ ] Приложение развёрнуто на HTTPS домене (GitHub Pages, Netlify, Vercel или свой сервер)
- [ ] Доступ администратора в Bitrix24
- [ ] Webhook URL из приложения Bitrix24 (если требуется server-side логика)

**Пример URL приложения:** `https://username.github.io/bitrix-expo-app`

## 2. Установка приложения через администратор

### 2.1 Через OAuth (рекомендуется)

1. Откройте Bitrix24 админ-панель
2. **Настройки приложений** → **Установить приложение**
3. Нажмите **"Установить из маркетплейса"** или **"Установить вручную"**
4. Введите manifest URL (если есть) или следуйте шагам ниже

### 2.2 Установка placement-ов вручную

1. Перейдите в **Администратор** → **Настройки приложений**
2. Нажмите **"Внешние приложения"** (или **"Приложения"**)
3. Нажмите **"Установить приложение"**

**Заполните форму:**

| Поле | Значение |
|------|----------|
| **Название** | Календарь выставок |
| **Описание** | Аналитика по выставкам, сделкам и лидам |
| **ID приложения** | `bitrix-expo-calendar` |
| **Секретный ключ** | Сгенерируется автоматически (или создайте сами) |

## 3. Регистрация placement-ов

После установки приложения зарегистрируйте placement-ы. Есть два способа:

### Способ A: Через интерфейс администратора (проще)

1. В меню приложения нажмите **"Привязать placement-ы"** (Bind Placements)
2. Для каждого placement заполните:

#### Placement 1: Аналитика (главное меню)

```
Placement: CRM_ANALYTICS_MENU
Title: Календарь выставок
Handler: https://YOUR_DOMAIN/calendar
Access: Public
```

#### Placement 2: Вкладка на сделке (опционально)

```
Placement: CRM_DEAL_DETAIL_TAB
Title: Выставка
Handler: https://YOUR_DOMAIN/deal-tab
Access: Public
```

#### Placement 3: Вкладка на лиде (опционально)

```
Placement: CRM_LEAD_DETAIL_TAB
Title: Выставка
Handler: https://YOUR_DOMAIN/lead-tab
Access: Public
```

### Способ B: REST API (для автоматизации)

Используйте метод `placement.bind`:

```bash
# Получить webhook URL приложения
curl -X POST https://your-portal.bitrix24.ru/rest/placement.bind \
  -H "Content-Type: application/json" \
  -d '{
    "PLACEMENT": "CRM_ANALYTICS_MENU",
    "HANDLER": "https://YOUR_DOMAIN/calendar",
    "TITLE": "Календарь выставок",
    "DESCRIPTION": "Аналитика по выставкам"
  }'
```

## 4. Проверка и тестирование

### Проверить установку

1. Откройте **CRM** → **Аналитика** (или **Настройки приложений**)
2. Должно появиться **"Календарь выставок"** (или ваше название)
3. Нажмите на него

### Первый запуск

1. Приложение откроется в iframe
2. Если нет данных:
   - Убедитесь, что в smart-process 1050 есть выставки
   - Проверьте права пользователя на чтение CRM
   - Откройте консоль браузера (F12) и посмотрите ошибки

### Диагностика

В самом приложении есть встроенная диагностика:
- Откройте приложение
- Скролльте вниз на странице Gantt
- **"Диагностика Gantt"** показывает статус всех API запросов

## 5. Переменные конфигурации (опционально)

Если нужно изменить поведение приложения, добавьте параметры placement-а:

```json
{
  "PLACEMENT": "CRM_ANALYTICS_MENU",
  "HANDLER": "https://YOUR_DOMAIN/calendar?compact=1",
  "PARAMS": {
    "SHOW_FILTERS": "1",
    "DEFAULT_MONTH": "current"
  }
}
```

## 6. Troubleshooting

### ❌ Приложение не показывается в Аналитике

**Решение:**
1. Очистите кэш браузера (Ctrl+Shift+Del)
2. Перезагрузитесь (F5)
3. Проверьте права администратора
4. Убедитесь, что placement зарегистрирован правильно

### ❌ "Доступ запрещён" или "Вы не можете просматривать это приложение"

**Решение:**
1. Проверьте, что HTTPS домен доступен
2. Убедитесь, что placement HANDLER URL начинается с https://
3. Проверьте CORS заголовки (должны быть разрешены от Bitrix24)

### ❌ "BX24 не определён" (только в демо-режиме)

**Решение:** Это нормально — приложение работает только внутри Bitrix24 iframe. Откройте из меню Bitrix24.

### ❌ Данные не загружаются / таймауты API

**Решение:**
1. Проверьте права пользователя на чтение CRM
2. Откройте диагностику (внизу Gantt)
3. Посмотрите, какой запрос падает
4. Может быть, у вас слишком много выставок (>500) — этот лимит настраивается в коде

## 7. Продвинутая настройка

### Поменять stage IDs для графика застройки

Если в вашем Bitrix24 используются другие stage IDs (не 8, 9, WON), пересоберите приложение:

```bash
VITE_BUILD_SCHEDULE_STAGE_IDS=10,11,CUSTOM npm run build
```

Затем развёрните новую версию.

### Кастомный Bitrix24 портал

Если приложение работает через proxy или нестандартный домен:

```bash
VITE_BITRIX_PORTAL_URL=https://custom-portal.bitrix24.ru npm run build
```

## 8. Обновление приложения

Когда вы пушите новые изменения:

1. GitHub Actions автоматически пересобирает и развёртывает на GitHub Pages
2. Или вы пересобираете вручную: `npm run build` и загружаете на Netlify / Vercel
3. Bitrix24 автоматически обновит приложение при следующем открытии (благодаря кэшированию Vite)

**Форс-обновление в браузере:** Ctrl+Shift+Del (очистить кэш) → F5

## 9. Удаление приложения

1. **Администратор** → **Настройки приложений** → **Внешние приложения**
2. Нажмите на приложение → **"Удалить"** (или **"Отвязать"**)
3. Подтвердите
